## Medium Findings
Count: 2

1. **[Vulnerability] Shell variable interpolation in audit event JSON construction** -- `skills/ship/SKILL.md` (multiple lines: 334, 989, 1024, 1197, 1243, 1346). The `/ship` skill constructs JSON strings for `emit-audit-event.sh` by interpolating shell variables directly into double-quoted strings (e.g., `"${CODE_REVIEW_VERDICT}"`, `"${SECURE_REVIEW_GATE_VERDICT:-not-run}"`). If a coordinator-controlled variable were to contain characters like `"` or `\`, this would produce malformed JSON. **Mitigating factor:** All interpolated values are generated by the coordinator itself from a fixed set of known values (e.g., `PASS`, `FAIL`, `BLOCKED`, `not-run`). The `emit-audit-event.sh` helper additionally uses `python3 json.loads()` to parse the partial event, which would reject malformed JSON and fall back to an empty dict. The values are never sourced from external/user input. **Severity: Medium** (defense-in-depth concern; not exploitable in current architecture).

2. **[Vulnerability] Test HMAC key is a predictable pattern** -- `scripts/test-integration.sh` line 153. The test uses `TEST_HMAC_KEY="abcd****6789"` (repeating hex pattern). This is clearly a test-only value and is never used in production. However, if this pattern were accidentally copied into a production configuration, the key would be trivially guessable. **Mitigating factor:** The key is used only in the integration test (Test 7) and is deleted after the test completes. The `.gitignore` excludes `.ship-audit-key-*` files. **Severity: Medium** (low risk, but the test could document the key as intentionally weak).

The cleanup function removes `/tmp/integration-smoke-test`, `$DEPLOY_DIR/smoke-coord`, and `$DEPLOY_DIR/smoke-pipe` using `rm -rf`. All paths are hardcoded constants derived from `$HOME/.claude/skills/` and `/tmp/`. No user-controlled input influences these paths. **Severity: Low** (safe usage pattern; no path traversal risk).
