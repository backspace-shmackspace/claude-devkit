**Stale MCP integration point in Workspaces section (CLAUDE.md line 720):**
The "Integration Points" bullet in the Workspaces Architecture section still states "Skills can invoke base agents via MCP." This was true prior to v3.0.0 but is now architecturally incorrect. The new pattern — established across all four core skills — is that skills check for local `.claude/agents/` project agents via Glob and fall back to `Task` subagents with `subagent_type=general-purpose`. MCP is no longer in the invocation chain.

**Stale artifact filename in README.md (line 231):**
Commit `0f81ab8` explicitly renamed the security scan output artifact from `audit-[timestamp].hardener.md` to `audit-[timestamp].security.md`. The commit message states: "rename artifacts from *.hardener.md to *.security.md." CLAUDE.md was correctly updated in the same commit (diff confirms the change at line 269 in CLAUDE.md). README.md was not updated and still references the old `*.hardener.md` filename, which will no longer be produced by `/audit`.

**Journal biweekly-update entry type (CLAUDE.md line 94, contrib registry):**

The Contrib Skill Registry in CLAUDE.md at line 94 describes the journal skill's purpose using an enumeration of entry types and does not include `biweekly-update`. The entry type has meaningful differentiation (auto-drafting from journal history, PSLT audience context, `deliverables/` directory output) that warrants mention in the registry description.
