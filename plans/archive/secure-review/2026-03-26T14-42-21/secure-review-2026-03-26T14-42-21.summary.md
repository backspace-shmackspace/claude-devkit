## High Findings
Count: 5

1. **[Vuln H-1 / Dataflow H-3 / Authz H-2] Shell injection via unsanitized `sed` substitution** -- `generators/generate_senior_architect.sh:297-298`. User-supplied `PROJECT_TYPE` is interpolated directly into `sed` replacement expressions without escaping. Allows content injection into generated agent files; on GNU sed, potentially command execution. Found independently by all three scans. **Remediation:** Deprecate shell script in favor of Python equivalent, or escape sed metacharacters.

2. **[Vuln H-2] Python `str.format()` / `format_map()` with user-controlled input** -- `generators/generate_senior_architect.py:344`, `generators/generate_skill.py:185`. Format string directives in user input could leak object attributes via `{__class__.__init__.__globals__}`. **Remediation:** Switch to `.replace()` (as `generate_agents.py` already does).

3. **[Dataflow H-1] `/secrets-scan` writes unredacted file contents to `./plans/` before scanning** -- `skills/secrets-scan/SKILL.md:73-106`. Raw file content (including potential secrets) is written to `plans/secrets-scan-*.scan-target.txt` before pattern detection runs. Archive failure is silently swallowed (`|| true`), leaving unredacted content on disk. **Remediation:** Write scan target to `/tmp/`, delete immediately after scanning, never archive raw content.

4. **[Dataflow H-2] Hardcoded user identity and employer in committed source files** -- Full name, employer (Red Hat), internal GitLab hostname (`gitlab.cee.redhat.com`), and username appear in CLAUDE.md, README.md, contrib skills, and templates. Repository is public on GitHub since 2026-03-25. Enables reconnaissance, social engineering. **Remediation:** Replace with GitHub handle or placeholders; generalize employer references.

5. **[Authz H-1 / Vuln M-1] Missing path traversal sanitization in `deploy_skill()` and `deploy_contrib_skill()`** -- `scripts/deploy.sh:19-47`. Unlike `undeploy_skill()` which validates against `../`, `/`, and `-*`, the deploy functions accept raw user input. Crafted skill name like `../../.ssh` could read/write arbitrary directories. **Remediation:** Extract validation from `undeploy_skill()` into shared function; apply to all three.
