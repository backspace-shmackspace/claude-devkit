## Path Resolution Changes (Detailed Analysis)

The commit 77898e8 made these changes:

**Before:**
```python
parser.add_argument(
    '--target-dir', '-t',
    default=str(Path.home() / 'workspaces' / 'claude-skills'),
    help='Target directory containing skills/ (default: ~/workspaces/claude-skills)'
)
```

**After:**
```python
parser.add_argument(
    '--target-dir', '-t',
    default=str(Path(__file__).resolve().parent.parent),
    help='Target directory containing skills/ (default: claude-devkit repo root)'
)
```

**Impact on Documentation:**

None. The documentation never said "generators default to `~/workspaces/claude-skills`". Instead:
- Installation workflow says "run `./scripts/install.sh`" which sets up env vars
- Manual installation shows setting `CLAUDE_DEVKIT` to actual clone location
- Deployment workflow says "`cd ~/projects/claude-devkit`" (example, not requirement)
- Generator examples show running from within repo: `gen-skill my-skill` (uses alias, works from anywhere)

The code change **removes a hardcoded assumption** that broke when claude-devkit was cloned to non-standard locations (like `~/projects/claude-devkit`). Documentation never made that assumption.
